# R42 — Class-membership derivation (anchoring on a supplied list)

This file contains the full normative procedure for `R42`. The compact row in `../common-rules.md` is only its routing summary.

## Full rule

| ID | Pattern | Procedure | Severity if missed |
|---|---|---|---|
| R42 | Class-membership derivation (anchoring on a supplied list) | **Fires when** (a) the task (plan or prompt) states or implies a *universal obligation* over a class — "every/all/each X must Y", or an invariant like "no Z without W" (step-up on all destructive routes, fail-closed on all rate limiters, a guard on a route-set, an enum applied to all consumers) — where the class Y's membership is discoverable by grepping a *defining primitive*; **OR (b)** a code-review finding is shaped as a *single instance* of such a class-invariant ("*this* route lacks control X" where X ought to hold universally) — the review's single-site report is a seed, not the set, and R42 fires to derive the rest even though neither plan nor prompt declared the class. Trigger (b) is the one the plan-declared reading of (a) silently misses; see the Trigger gate item 2 in Extended obligations. Does NOT fire for changes scoped to an explicitly enumerated, closed set the plan itself owns (cite the contract / `SCn`). **Clause ① (member-set derivation)**: the member-set of Y MUST be derived by grepping the primitive that *defines* Y across the whole codebase — NOT taken from the prompt's or an external review's enumeration. A prompt-, plan-, or external-review-supplied list is an *unverified hint*, never authoritative — treat it exactly as RS5 treats an externally-supplied security parameter. Declaring the invariant obligates deriving its member-set from code *in the same step*; every list-vs-code delta is a finding in its own right (not merely a prompt to look closer). **Clause ①a (primitive vs symptom)**: grep the operation whose *unguarded execution is the bug* (the defining primitive), not a *symptom* that merely co-occurs (a shared config value, an isolation-level annotation, an error string) — a symptom grep yields an incomplete, unstable member-set that grows by accretion every round (the `isolationLevel`-vs-`count→cap→create` TOCTOU miss that expanded 1→5→6→10→17). **Clause ①b (one expansion ⇒ re-derive)**: the first time the member-set expands after derivation, treat it as proof the anchor was a symptom — discard and re-derive from a corrected primitive, do not append-and-continue. **Clause ③ (structurally-invisible paths)**: when deriving Y's membership, include *indirect* members — writes/operations that reach Y via parent cascade, raw SQL bypassing the ORM/guard layer, or an aliased/wrapped service caller — not only direct primitive call sites. A destructive or state-changing path the class-guard cannot structurally observe is a member it does not cover. **Symmetric-counterpart sub-clause**: mirrored handler families are another structurally-invisible member source — when the codebase exposes the same operation through paired or parallel surfaces (two ownership scopes, two privilege levels, two access modes over the same underlying resource) and a control is applied to one surface, the mirror surface's equivalent operations are members of Y too. Deriving the member-set from the defining-primitive grep must span every parallel surface; "applied the control to all of surface A" silently omits surface B's operations that act on the same class of data. For every control applied to one side of a known symmetry, enumerate and check the mirror side, or document the exclusion with a reason. **Distinct from**: R3 (propagates from an already-identified instance — R42 questions the seed set itself), R17/R22 (gated on a *new helper*; R42's control often reuses an existing primitive with no new helper), R12 (enum-group registration; R42 generalizes the same derive-from-code discipline to control-application). **Complementary to** RS2/RS3/R4 (which fire on *new* boundaries): R42 covers retrofitting a control across an *existing* class, where the failure is a pre-existing member missed during derivation — RS2's new-route trigger never fires for Y's un-retrofitted existing members. **Interaction with R38**: a class-guard that misses a member is fail-open by *omission* (R38 covers fail-open by *late async write*) — same Critical floor, different mechanism. **Fingerprint/attestation input-set sub-clause (mined extension)**: a cache key, change-fingerprint, or "nothing relevant changed" attestation that gates SKIPPING a verification step is a universal claim over a *data-input* class — its hashed-input member-set must be derived from the guarded process's actual READ SURFACE (every path-like input it can read: tracked and ignored and untracked files, content as transformed by any VCS filters, every filesystem object type the walk can meet), never from the hasher's convenience set (what the VCS diff surfaces cheaply). Any input type the fingerprint cannot safely and completely represent (special files, unreadable objects, forgeable record encodings) must FAIL CLOSED — abort the skip and run the full verification, never record a hit — with one red fixture per input class proving the bypass is caught. A stale cached "pass" surviving a real change is the fail-open direction of this rule applied to data instead of code. **Reviewer action / verification**: recompute the set — `grep -rlE '<primitive that defines Y>' <code roots>` → set A; the diff's touched-member set → set B; `A \ B` non-empty is the finding. Also grep the cascade/relation definition and the raw-query surface for clause ③, not just the ORM method name. **Detection**: human-review (the class boundary lives in the plan/prompt via trigger (a), or in the shape of a code-review finding via trigger (b) — not in the diff's syntax) — but the A\B set-difference is a `check-control-coverage.sh` candidate (not yet implemented). | Major by default; **Critical when the un-derived member means a security control (step-up, authz guard, rate limit, fail-closed gate, revocation) does not cover a member it was declared to cover — fail-open, same direction R38 Part 2 flags Critical** |
